mysql索引学习笔记

发表于 2016-09-09   |   分类于 技术

MySQL官方对索引的定义为：索引（Index）是帮助MySQL高效获取数据的数据结构。简单通俗的比喻一下，索引就好比一本书的目录，能够快速的定位到你想查找的位置上去。

索引的数据结构

mysql的索引采用了一种叫做B+TREE的数据结构。它是一个N叉数树，一种多路搜索树。B+TREE是B-TREE的一个变种。B+TREE的具体内同就不介绍了。

索引又分为聚簇索引和非聚簇索引（对InnoDB而言）

  • 聚簇索引：在该索引实现方式中B+Tree的叶子节点上的data就是数据本身，每个表只能拥有有一个聚簇索引。
  • 非聚簇索引（二级索引）：非聚簇索引就是指B+Tree的叶子节点上存储的是相应记录的主键值。

索引的使用

mysql 的索引又分为单列索引和复合索引。

java的栈与堆

发表于 2016-08-21   |   分类于 技术

java的内存主要分成栈（stack）和堆（heap）两部分。在函数中，一些基本类型的变量和对象的
引用变量都会分配在栈内存中，堆内存主要用于存放由new创建的对象和数组。从堆和栈的功能和作
用来通俗的比较,堆主要用来存放对象的，栈主要是用来执行程序的

栈的优缺点

栈的优点是对于小块内存来说，它更容易命中cache，所以效率会比堆更高，而且栈之间可以共享数据。栈的缺点是，存在栈中的数据大小与生存期必须是确定的，缺乏灵活性。

堆的优缺点

堆内存用于存放由new创建的对象和数组。在堆中分配的内存，由java虚拟机自动垃圾回收器来管理
。堆的优点在于编译器不必知道要从堆里分配多少存储空间，也不必知道对象的生命周期,因此,用堆保存数据时会得到更大的灵活性。但是与栈相比，堆的分配和释放比栈更加复杂，而且堆上的内存很有可能不在寄存器或者CUP缓存中，访问命中率就低，所以效率也就会低。

JVM中的堆和栈

JVM是基于堆栈的虚拟机。JVM为每个新创建的线程都分配一个堆栈。也就是说,对于一个Java程序来说
，它的运行就是通过对堆栈的操作来完成的。堆栈是操作系统在建立某个线程并为这个线程建立的存储区域，该区域具有先进后出的特性。
每一个Java应用都唯一对应一个JVM实例，每一个实例唯一对应一个堆。应用程序在运行中所创建的所
有类实例或数组都放在这个堆中，并由应用所有的线程共享。Java中分配堆内存是自动初始化的。Java中所有对象的存储空间都是在堆中分配的，但是这个对象的引用却是在堆栈中分配,也就是说在建立
一个对象时从两个地方都分配内存，在堆中分配的内存实际建立这个对象，而在栈中分配的内存只是
一个指向这个堆对象的指针(引用)而已。

java垃圾回收总结

发表于 2016-08-20   |   分类于 技术

前几天看了有关于java的垃圾回收机制方法的文章，感觉自己有了一点收获，在遗忘之前赶紧记录下来。

java不像c和c++那样需要程序员自己去分配和回收内存，java是虚拟机动态的分配内存以及回收内存的，java的内存回收装置叫做垃圾收集器。

虚拟机去回收垃圾的时候肯定需要知道哪些对象需要回收，哪些又不需要回收。这里java就运用到根搜索算法。

根搜索算法：

把一些对象作为“GC Root”，也就是起点，从这个起点开始向下搜索，搜索的路径就叫做引用链。如果一个对象到起点没有引用链，那么这个对象就是不可用的，需要被回收的。GC Root包括：

  • 虚拟机栈（栈帧中的本地变量表）中的引用的对象。
  • 方法区域中的类静态属性引用的对象。
  • 方法区域中常量引用的对象。
  • 本地方法栈中JNI（Native方法）的引用的对象。

java虚拟机一般采用的是分代回收机制，java堆分为新生代和老年代，再根据不同的年代采用不同的收集算法。新生代的主要特点就是大批对象都是可被回收的，只有少数对象还存活着。老年代的特点就是对象的存活率都比较高。

在回收新生代的时候我们采用的是复制算法。

下面简单介绍一下复制算法：
该算法主要把内存划分为一块Eden空间和两块Survivor空间，在虚拟机中他们的默认比值为8：1，也就是说Eden的空间很大。每次都只会使用一块Eden和一块Survivor，当回收的时候，先会把Eden和一块Survivor上面存活的对象拷贝到另外一块Survivor上面，最后再清理掉刚才的Eden和被使用过的Survivor。

在回收老年代的时候我们用的是标记-清除，或者标记-整理算法。

标记清除算法比较简单，但是容易产生内存碎片，在后续的运行过程中可能会造成无法分配大对象。主要是因为标记清除算法首先会把要回收的对象标记出来，然后再清楚点所有被标记的对象，这些对象很可能在内存中都是不连续的。标记整理算法对前面的标记清除进行了优化，和前面一样，它也是想把要清除的对象进行标记，然后把所有存活的对象都移到一端，然后再把这个端意外的对象清除掉。

下面说一下常用的垃圾收集器。

在新生代中最常用的是ParNew收集器。在收集新生代是采用的是复制算法，这个收集器是使用多线程进行垃圾收集。一般用在多核CPU环境下，它一般与CMS收集器配合使用。
老年代中常用的是CMS收集器，它使一种以获取最短回收停顿时间为目标的收集器。它的收集过程一共分为四个步骤：

  • 初始标记
  • 并发标记
  • 重新标记
  • 并发清除

1，3是阻塞的，需要暂停所有其他线程。而2和4是并发完成的，不需要暂停其他用户线程。

其他的收集器还包括Serial（新生代），Parallel Scavenge（新生代），Serial Old（老生代），Parallel Old（老生代），G1等。

mac下安装erlang

发表于 2016-08-19   |   分类于 技术

  • 下载erlang源码

    地址http://www.erlang.org或者https://github.com/erlang/otp

  • 将源码进行解压缩

    tar -zxf otp_src_17.4.tar.gz

  • 进入到解压缩的文件夹

    cd otp_src_17.4

  • 配置环境变量

    vi ~/.bash.profile

    export ERL_TOP=pwd

    （pwd是获取当前的绝对路径，注意一定要是erlang源码的地址）

    export LANG=C

  • 编译与安装

    make

    sudo make install

  • 安装目录是：/usr/local
